Learn About APME

Intelligent, automatic algorithm selection backed by rigorous complexity theory.

C-Accelerated Core

All six algorithms are implemented in optimised C with SIMD-friendly memchr calls. Python bindings via ctypes keep startup overhead near zero.

Automatic Selection

APME analyses pattern length, text size, alphabet cardinality, and repetitiveness, then selects the optimal algorithm — no configuration needed.

Worst-Case Guarantees

Every exact-match algorithm provides a proven O(n) or O(n+m) worst-case bound. No adversarial input can trigger quadratic behaviour.

APME's heuristic selector evaluates the following decision table in order:

# Condition Algorithm Selected
2 Pattern length m ≤ 2 FlowScan
2.5 m ≤ 64 and ASCII-only pattern BitAnchor
3 More than 1 pattern WebScan
4 Alphabet cardinality σ ≤ 4 (binary/DNA) FlowScan
5 Repetitiveness ratio > 70% FlowScan
6 m > 10 and σ > 10 and n > 5,000 SkipStride
7 m ≤ 10 and n > 100,000 TwinHash
8 Default (general text) SkipStride

TierMatch (approximate) and WebScan (multi-pattern) are also available as manual selections.
